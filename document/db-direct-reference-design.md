# DB 直接参照機能 設計メモ

## 1. 背景と目的

### 1.1 既存挙動

現在の CSV ビューアは、抽出済み CSV を読み込み、一覧から選択したレコードについて以下を表示する。

- 左ペイン: フィルタと抽出済みレコード一覧
- 右ペイン: 選択レコードの属性情報と `paragraph_text_tagged` ベースの強調表示本文

現状の本文表示は CSV に保存されたテキストに依存しており、元データベースの原文や前後コンテキストは参照しない。

### 1.2 追加機能の目的

本機能の目的は次の 2 点である。

1. CSV に格納されたテキストが、元データベース上の段落テキストと整合しているか確認する
2. 抽出された段落の前後コンテキストを確認する

初期実装では、分析単位を段落とみなし、段落単位の確認機能を優先する。文単位の参照は将来拡張として扱う。

## 2. 参照対象

### 2.1 DB ファイル

- パス: `asset/ordinance_analysis3.db`

### 2.2 参照テーブル

- テーブル: `analysis_paragraphs`

### 2.3 確認済みカラム

- `paragraph_id`
- `run_id`
- `document_id`
- `paragraph_no`
- `paragraph_text`
- `created_at`

### 2.4 キー設計

- 中心段落の特定キー: `paragraph_id`
- 前後段落の取得キー: `document_id` と `paragraph_no`

注意:
`paragraph_id` は主キーであるが、前後段落の取得に `paragraph_id ± n` を使ってはならない。前後文脈は同一文書内で保証する必要があるため、`document_id` 固定かつ `paragraph_no` 範囲で取得する。

## 3. 機能要件

### 3.1 必須要件

1. 右ペインに DB 参照への入口を追加する
2. 入口操作により、DB 参照用の別ウィンドウを開ける
3. 選択中 CSV レコードの `paragraph_id` を使い、`analysis_paragraphs` から中心段落を取得できる
4. 取得した中心段落から `document_id` と `paragraph_no` を求め、同一文書の前後 2 段落を取得できる
5. DB 参照ウィンドウには、中心を含む最大 5 段落を表示できる
6. DB 参照ウィンドウには、現在の中心段落がどれか明示表示される
7. CSV の表示テキストと DB の段落テキストを比較できる
8. 前後移動のための操作を用意する
9. 文書の先頭または末尾では、存在する段落だけを表示する
10. DB 参照に失敗した場合、理由を UI 上で明示する

### 3.2 読み取り専用要件

1. DB に対して更新処理を行わない
2. DB 参照は表示専用とする
3. CSV 再生成、抽出条件再実行、分析再計算は本機能の対象外とする

### 3.3 運用要件

1. DB 参照機能は補完機能とし、DB が利用できなくても既存の CSV ビューアは動作する
2. DB ファイルが存在しない場合、参照ボタン押下時にエラー表示する
3. `paragraph_id` に対応するレコードが DB に存在しない場合、参照不能であることを明示する

## 4. UI 仕様

### 4.1 メイン画面

対象:
- 既存の右ペイン

追加要素:
- `DB参照` ボタン

配置案:
- 現在の詳細ヘッダ付近、または属性情報ブロックの上部

動作:
- レコード未選択時は無効化
- 選択レコードがある場合に押下可能
- 押下時に別ウィンドウを開く

### 4.2 DB 参照ウィンドウ

表示内容:

1. ヘッダ領域
- `paragraph_id`
- `document_id`
- `paragraph_no`
- DB ファイル名

2. 比較領域
- CSV 側テキスト
- DB 側中心段落テキスト
- 不一致時は視覚的に分かる表示

3. コンテキスト領域
- 前 2 段落
- 中心段落
- 後 2 段落

表示ルール:
- 中心段落は背景色やラベルで強調する
- 各段落には最低限 `paragraph_no` を表示する
- 段落本文は折り返し表示する

4. 操作領域
- `前へ` ボタン
- `次へ` ボタン
- 必要なら現在位置表示

### 4.3 前後移動仕様

初期案:
- `前へ`: 中心段落を 1 段落前へ移動
- `次へ`: 中心段落を 1 段落後へ移動

移動後:
- 新しい中心段落を基準に、再度前後 2 段落を取得して表示する

境界条件:
- 先頭段落では `前へ` を無効化
- 末尾段落では `次へ` を無効化

備考:
- スクロールでもよいが、MVP では `前へ` / `次へ` ボタンの方が実装と挙動の明確性が高い

## 5. データ取得仕様

### 5.1 初回表示

1. CSV 選択レコードから `paragraph_id` を取得する
2. `analysis_paragraphs` から中心段落を 1 件取得する
3. 取得した中心段落の `document_id`, `paragraph_no` を使って前後段落を取得する
4. `paragraph_no - 2` から `paragraph_no + 2` の範囲を同一 `document_id` 内で検索する
5. `paragraph_no` 昇順で表示する

### 5.2 前後移動

1. 現在中心の `document_id`, `paragraph_no` を保持する
2. 前後移動時は中心 `paragraph_no` を更新する
3. 更新後の中心段落と前後 2 段落を再取得する

### 5.3 不一致判定

初期実装での比較対象:
- CSV 側: 現在右ペインに表示している段落テキスト
- DB 側: `analysis_paragraphs.paragraph_text`

判定方針:
- まずは単純文字列比較でよい
- 差分ハイライトは将来拡張とする

## 6. エラー・例外仕様

1. DB ファイルが開けない
- ウィンドウ内にエラーメッセージを表示する

2. `paragraph_id` が見つからない
- 「対応する DB 段落が見つかりません」と表示する

3. 前後段落の一部が存在しない
- 取得できた分だけ表示する
- これは正常系として扱う

4. CSV の `paragraph_id` が数値として解釈できない
- DB 参照不可として扱う
- 参照不能の理由を表示する

## 7. 初期スコープ外

- 文単位参照
- 段落と文を切り替える共通コンポーネント化
- DB 上の他テーブル参照
- 抽出条件や分析条件の再実行
- テキスト差分の精密ハイライト

## 8. 将来拡張案

1. 段落/文の参照単位を切り替え可能にする
2. 差分表示を導入する
3. 同一文書全体を開くビューを追加する
4. DB パスを UI から切り替えられるようにする

## 9. 実装手順

### Step 1. DB 参照レイヤ追加

- SQLite 接続用の小さな読み取りモジュールを追加する
- `paragraph_id` から中心段落を取得する関数を作る
- `document_id + paragraph_no` から前後 2 段落を取得する関数を作る

成果物:
- DB 参照用のデータ型
- 読み取り専用クエリ関数

### Step 2. App 状態追加

- DB 参照ウィンドウ開閉状態を追加する
- 現在の DB 参照結果を保持する状態を追加する
- 参照中の中心 `paragraph_id` または `paragraph_no` を保持する
- エラー状態を保持する

成果物:
- `App` の DB 参照状態

### Step 3. 右ペインに入口追加

- `DB参照` ボタンを追加する
- レコード未選択時の無効化を実装する
- ボタン押下時に DB 参照処理を開始する

成果物:
- メイン画面からの導線

### Step 4. 別ウィンドウ実装

- DB 参照用ウィンドウを追加する
- 比較領域とコンテキスト領域を実装する
- 中心段落の強調表示を実装する

成果物:
- DB 参照ウィンドウ

### Step 5. 前後移動操作追加

- `前へ` / `次へ` ボタンを追加する
- 文書先頭/末尾での無効化を実装する
- 移動時に中心段落と表示内容を再取得する

成果物:
- コンテキストナビゲーション

### Step 6. エラー表示と例外処理

- DB 未存在
- 参照失敗
- `paragraph_id` 不正
- 対応段落なし

を UI 上で明示する

### Step 7. 動作確認

- 中心段落の一致確認
- 前後 5 段落表示
- 先頭/末尾境界
- DB なし時の表示
- CSV と DB 不一致時の表示

## 10. 実装順の推奨

1. DB クエリ関数
2. `App` 状態追加
3. 右ペイン入口
4. DB ウィンドウ本体
5. 前後移動
6. エラー表示
7. 調整

この順序で進めると、UI を先に複雑化させず、データ取得の正しさを先に固められる。
